## C-1. ネットワーク接続問題

### 何が問題か

- 多くのスタックチャンは **ネット接続を前提に会話**（クラウドのLLM/音声認識/音声合成を利用）しています。当日、来場者と楽しく交流するには **Wi-Fi共有がほぼ必須** です。
- **人が多い会場はネットワークが混雑・輻輳しやすく**、つながらない／遅延が大きいといったトラブルが起きがちです。本番で困らないよう **事前に解決** しておきたい点です。

### 影響

- 会話デモが止まる／極端に遅くなる → 体験の質が大きく低下
- 複数台のスタックチャン＋来場者スマホで **同時接続数・帯域** を圧迫

### 対策案

| 対策 | 内容 | 備考 |
|---|---|---|
| 会場Wi-Fiの事前確認 | 提供有無・帯域・**同時接続上限**・5GHz対応を主催/会場に確認 | まず確認すべき最重要事項 |
| モバイルルーター持込 | 自前回線を用意し会場網に依存しない | 大手町＝地下/混雑で電波状況の事前テスト推奨 |
| 有線接続の確保 | 可能なら有線LAN/USBテザリングで安定化 | ブース位置で配線可否を確認 |
| **オンデバイスLLMで依存を低減** | M5Stack **LLM Module** 等で **ローカル会話**（ネット不要・低遅延）| → [アイディア I-1](/ideas#i-1-llm-module-デモ) と直結 |
| ネット不要の動作も用意 | 表情・モーション・定型応答など **オフラインでも成立** する演出を準備 | 通信断時のフォールバック |

### 要確認（主催・会場へ）

- [ ] 会場Wi-Fiの提供可否／SSID・帯域／**同時接続数の上限**
- [ ] ブース位置（1F コミュニティショーケース）での電波状況
- [ ] モバイルルーター／有線の持込可否
- [ ] 来場者にもWi-Fiを使わせる場合の運用（QR掲示など）
